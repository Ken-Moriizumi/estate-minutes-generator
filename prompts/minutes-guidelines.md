# 議事録作成ガイドライン

このドキュメントは、AIが議事録を生成する際の指針を定義します。

## ⚠️ 最重要ルール

**絶対に守るべき記載方式：**

1. **会話形式を使わない**
   - ❌ NG: 「〇〇取締役は『〜』と述べた」
   - ❌ NG: 「〇〇からは『〜』との意見が出た」
   - ❌ NG: カギ括弧（「」）を使った発言の引用
   - ✅ OK: 「検討結果: 〜である。〜と判断される。」

2. **個人名を出さない**
   - ❌ NG: 「裕美取締役からは〜」
   - ❌ NG: 「山田社長は〜と指摘した」
   - ✅ OK: 「検討結果」として全体の結論のみ記載

3. **必ず「検討結果:」で始める**
   - 各物件の議事内容は必ず「検討結果:」で始まる1段落で完結させる
   - 複数の文を連ねて、検討の内容と結論を記載

## 参加者プロファイル

**注意**: 参加者プロファイルは議事録生成の参考情報ですが、実際の議事録では個別の発言は記載しません。全体の検討結果として統合します。

### 検討の視点（参考）

会議では以下のような視点から物件が検討されます：

**投資価値の分析**
- 利回り、キャッシュフロー、ROI
- 価格の妥当性
- 収益性の評価

**リスク分析**
- 立地条件と賃貸需要
- 空室リスク
- 修繕費用
- 金利上昇リスク

**実務的な懸念**
- 管理の手間
- 入居者確保の見込み
- 税務面の考慮

## 議事録作成の観点

### 1. 目的
- **税務調査対応**: 実質的な業務会議が行われたことを証明
- **会議費・社内交際費の適正な経費計上**

### 2. 内容要件
- **実質性**: 形式的な議事録ではなく、実際に議論が行われた様子を記載
- **具体性**: 物件の具体的なスペックや条件に言及
- **多様性**: 参加者ごとに異なる視点・意見を記載
- **現実性**:
  - 全員が完璧に理解している必要はない
  - 非常勤役員は「よく分からなかった」という感想も可
  - 専門用語の理解度に差があることを反映

### 3. ボリューム
- **A4用紙1〜2枚程度**（800〜1,600文字）
- 物件数が多い場合は2枚に近づける
- 物件数が少ない場合でも1枚は確保

### 4. 議題の設定
- 物件ごとに議題を設定
- 議題タイトルは「[物件名/所在地]の投資検討」「[物件名/所在地]の収益性分析」などバリエーションをつける

### 5. 物件情報の記載
- ＜物件情報＞セクションで簡潔に要約
- 必須項目: 築年数、所在地、価格
- 任意項目: 利回り、構造、面積、その他特記事項
- メール本文から抽出できる情報を使用

### 6. 検討結果の記載ルール（最重要）

**必須フォーマット:**
```
検討結果: [物件の評価]。[立地の分析]。[リスク評価]。[結論]。
```

**記載方法:**
- 必ず「検討結果:」で始める
- 1つの段落にまとめる
- 会議での議論を統合した結論として記載
- 個人名や発言者を特定しない
- カギ括弧（「」）を使わない
- 客観的・分析的な文体

**良い例:**
```
検討結果: 価格は500万円以下と安価だが、立地が武蔵嵐山駅徒歩3100m（約39分）と極めて悪く、賃貸需要が見込めない。高い空室リスクが想定され、投資額に見合う収益率を確保できないため、見送るべきである。
```

**悪い例（絶対NG）:**
```
検討結果: 価格は380万円と安価である。
裕美取締役からは「駅からとても遠いので、入居者がいるのか心配です。」との意見が出た。
```
→ 個人名と会話形式は絶対に使わない

### 7. 結論の書き方
- 各物件について簡潔に結論を記載
- 「〜することとした」「〜を確認する」など、今後のアクションを含める
- 全体としての総括も追加

### 8. 文体
- 「です・ます調」ではなく「だ・である調」を使用（議事録として）
- 参加者の発言は「」で囲む
- 敬語は使用するが、過度に丁寧にしない

## 生成時の注意事項

1. **メール本文をよく読む**: メールに記載された物件情報を正確に把握する
2. **情報の捏造禁止**: メールにない情報は推測で追加しない
3. **バランス**: 代表取締役社長だけが詳しく、他の参加者は簡潔に、というバランスを保つ
4. **自然な流れ**: 会議の自然な流れを意識（全員が同じ論調にならない）
5. **ページ数調整**: 物件数に応じてA4 1〜2枚になるよう調整
